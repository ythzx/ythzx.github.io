2 异步非阻塞实现并发

现在主流的实现并发的方案就是异步非阻塞，异步非阻塞是实现高性能的关键，异步是回调，非阻塞是不等待

  • 异步 回调 执行完任务后执行回调函数
  • 非阻塞 不等待(针对的是一个socket)

默认的socket连接的时候是阻塞的，等待发消息然后等待数据返回，如何设置成非阻塞的？

在创建socket对象的时候，设置setblocking（False）就设置成非阻塞，但是recv的时候，数据不可能立即接收到，因此会报错。非阻塞是发送数据和接收数据都不等待。

异步非阻塞结合

异步和非阻塞结合起来才有意义，最终就是开始是把conn一直发送，此时不发送数据，当有连接成功后，返回谁连接成功了，然后给其发送数据，最后谁返回数据了，返回告知。(Http是一次请求，一次响应)

2.1 协程

gvent本身是没有协程的，内部依赖greenlet，greenlet是真正实现协程的 libevent

greenlet 实现了协程 不能实现遇到IO阻塞自动切换，需要自己手动切换。与异步非阻塞没有关系。

gevent 实现了遇到IO阻塞的时候切换,gevent 内部把socket设置成了非阻塞状态
